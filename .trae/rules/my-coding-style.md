USE snackCase for variables and functions naming
USE PascalCase for component or classes or Types naming

FOLLOW SOLID principles for code organization and design.
FOLLOW DRY (Don't Repeat Yourself) principle for code duplication.
FOLLOW KISS (Keep It Simple, Stupid) principle for code simplicity.
FOLLOW YAGNI (You Aren't Gonna Need It) principle for code complexity.
FOLLOW "Separation of Concerns" principle for code organization.
FOLLOW "Single Responsibility Principle" for code maintainability.

ALWAYS use `signals` for state management for angular projects by default.
ALWAYS use `standalone` components for better DX and tree-shakability in angular projects.
ALWAYS use `tailwind-variants` for custom styles when it is present in the project.
ALWAYS use `tailwindcss@v4` for styling in angular projects by default.
